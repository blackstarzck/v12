# TALKPIK AI Frontend Specification

이 문서는 TALKPIK AI를 새 프론트엔드로 개발할 때 따라야 할 기준입니다. 프론트엔드는 ReactJS로 만들고, 화면 상태는 Zustand로 관리하며, 기본 UI 구성요소는 shadcn/ui와 Tailwind CSS를 사용합니다. 디자인 방향은 프로젝트에 설치한 Supanova Design Skill을 기준으로 합니다.

## 1. 개발 결론

TALKPIK AI는 TOPIK 학습자가 매일 들어와 학습 현황을 확인하고, 문제를 풀고, 쓰기 피드백을 다시 확인하는 학습 앱입니다. 따라서 화면은 단순한 홍보용 랜딩 페이지가 아니라 반복 사용에 편한 학습 대시보드로 설계해야 합니다.

핵심 구현 기준은 다음과 같습니다.

- ReactJS: 화면을 컴포넌트 단위로 나누어 개발합니다. 컴포넌트는 버튼, 카드, 입력 폼처럼 재사용 가능한 화면 조각입니다.
- Zustand: 여러 화면에서 공유해야 하는 상태를 저장합니다. 상태는 현재 사용자, 학습 진행률, 풀이 중인 문제, 쓰기 답안, AI 튜터 패널 열림 여부 같은 앱의 현재 상황입니다.
- shadcn/ui: 버튼, 카드, 다이얼로그, 탭, 셀렉트 같은 접근성 있는 UI 기본 부품으로 사용합니다.
- Tailwind CSS: 여백, 색상, 글자 크기, 반응형 레이아웃을 클래스 이름으로 빠르게 지정하는 스타일 방식으로 사용합니다.
- Supanova Design Skill: 한국어 중심의 고급스러운 화면 밀도, 자연스러운 한국어 문구, 명확한 CTA, 과하지 않은 모션, 반복적이지 않은 레이아웃을 만드는 디자인 기준으로 사용합니다.

## 2. 설치된 디자인 스킬

Supanova Design Skill은 프로젝트 안에 설치되어 있습니다.

- `.codex/skills/taste-skill/SKILL.md`: 전체 디자인 엔진입니다. 프리미엄 랜딩 페이지, 한국어 타이포그래피, 색상, 레이아웃 기준을 정의합니다.
- `.codex/skills/soft-skill/SKILL.md`: 더 정교한 미감 기준입니다. 카드 구조, CTA 버튼, 모션, 여백, 한국어 문장 품질을 강화합니다.
- `.codex/skills/redesign-skill/SKILL.md`: 기존 화면을 개선할 때 사용하는 진단 및 리디자인 기준입니다.
- `.codex/skills/output-skill/SKILL.md`: 생략 없는 완성 산출물을 요구하는 기준입니다.

이 프로젝트에서는 Supanova의 단일 HTML 출력 규칙을 그대로 따르지 않습니다. 대신 ReactJS, Zustand, shadcn/ui, Tailwind CSS 구조 안에서 Supanova의 디자인 원칙을 적용합니다.

## 3. 기술 스택

### 3.1 ReactJS

ReactJS는 화면을 작은 컴포넌트로 나누어 조립하는 프론트엔드 라이브러리입니다.

권장 기준:

- TypeScript를 함께 사용합니다. TypeScript는 데이터 모양을 미리 정의해 실수를 줄이는 JavaScript 확장 언어입니다.
- 화면 단위 컴포넌트와 재사용 컴포넌트를 분리합니다.
- 한 컴포넌트가 너무 많은 일을 하지 않도록 데이터 표시, 사용자 입력, 레이아웃 책임을 나눕니다.
- 서버에서 받아온 데이터와 사용자가 화면에서 조작하는 임시 상태를 구분합니다.

권장 폴더 구조:

```text
src/
  app/
    routes/
    layouts/
  components/
    ui/
    app/
    learning/
    writing/
    mock-exam/
    ai-tutor/
  stores/
  hooks/
  lib/
  types/
  styles/
```

### 3.2 Zustand

Zustand는 전역 상태 관리 라이브러리입니다. 전역 상태는 여러 화면에서 함께 쓰는 앱의 기억장치라고 보면 됩니다.

필수 스토어:

- `useUserStore`: 사용자 이름, 플랜, 목표 급수, 언어 설정을 저장합니다.
- `useLearningStore`: 주간 학습 시간, 출석, 영역별 실력, 약점 정보를 저장합니다.
- `usePracticeStore`: 읽기와 듣기 문제 생성 조건, 현재 문제, 선택한 답, 정답 확인 상태를 저장합니다.
- `useWritingStore`: 쓰기 51번부터 54번까지의 답안 초안, 자동 저장 상태, 제출 상태를 저장합니다.
- `useFeedbackStore`: AI 쓰기 피드백 목록, 상세 피드백, 다시 보기 상태를 저장합니다.
- `useVocabularyStore`: 저장한 단어, 필터, 복습 상태를 저장합니다.
- `useMockExamStore`: 모의고사 설정, 타이머, OMR 답안, 응시 진행 상태를 저장합니다.
- `useAiTutorStore`: AI 튜터 패널 열림 여부, 현재 탭, 채팅 기록, 알림 상태를 저장합니다.
- `useUiStore`: 사이드바, 모바일 메뉴, 토스트, 모달 같은 공통 UI 상태를 저장합니다.

주의사항:

- 서버에서 다시 가져올 수 있는 데이터는 Zustand에 장기 저장하지 않습니다.
- 사용자가 입력 중인 답안처럼 잃어버리면 안 되는 임시 데이터는 자동 저장 기준을 둡니다.
- 스토어 이름은 기능 중심으로 붙입니다.
- 스토어는 화면을 직접 알지 않게 만듭니다. 화면은 스토어를 읽고 행동을 호출하는 역할만 합니다.

### 3.3 shadcn/ui

shadcn/ui는 완성된 UI 부품을 프로젝트에 복사해 커스터마이징하는 방식의 컴포넌트 모음입니다.

필수 컴포넌트:

- `Button`: 주요 행동 버튼입니다.
- `Card`: 학습 현황, 문제 유형, 피드백 요약을 묶는 박스입니다.
- `Dialog`: 확인, 제출, 삭제 같은 중요한 선택을 받을 때 사용합니다.
- `Sheet`: 모바일 메뉴와 AI 튜터 패널처럼 화면 옆에서 열리는 패널에 사용합니다.
- `Tabs`: 쓰기 피드백, AI 튜터 탭, 모의고사 결과 탭에 사용합니다.
- `Select`: TOPIK 단계, 문제 유형, 언어 선택에 사용합니다.
- `Textarea`: 쓰기 답안 입력에 사용합니다.
- `Progress`: 목표 달성률과 시험 진행률에 사용합니다.
- `Badge`: 상태, 난이도, 급수, 문제 유형 표시로 사용합니다.
- `Tooltip`: 아이콘 버튼의 의미를 짧게 알려줄 때 사용합니다.
- `Toast`: 저장 완료, 제출 완료, 오류 안내에 사용합니다.

shadcn/ui는 기본 부품 역할을 합니다. 실제 TALKPIK AI 화면에서는 `LearningCard`, `WritingFeedbackSummary`, `MockExamTimer`, `AiTutorPanel` 같은 도메인 컴포넌트로 한 번 더 감싸서 사용합니다.

### 3.4 Tailwind CSS

Tailwind CSS는 디자인 토큰과 레이아웃 규칙을 코드 안에서 일관되게 쓰기 위한 스타일 도구입니다.

필수 기준:

- 공통 색상, 글자 크기, 반경, 그림자, 여백은 Tailwind 설정으로 관리합니다.
- 한국어 문장은 줄바꿈이 어색하지 않도록 `break-keep` 또는 동일한 CSS 규칙을 적용합니다.
- 모바일 우선으로 작성하고 `md`, `lg`, `xl` 기준에서 넓은 화면을 확장합니다.
- 화면 높이는 모바일 브라우저 흔들림을 줄이기 위해 `100dvh` 기반을 사용합니다.
- 인터랙션 모션은 `transform`과 `opacity` 중심으로 처리합니다.

## 4. 디자인 방향

Supanova Design Skill을 TALKPIK AI에 맞게 적용한 디자인 방향은 다음과 같습니다.

### 4.1 전체 인상

TALKPIK AI는 시험 준비 서비스이므로 신뢰감, 집중감, 반복 사용성이 가장 중요합니다. 화면은 고급스럽되 과장된 광고 화면처럼 보이면 안 됩니다.

디자인 키워드:

- 프리미엄 학습 대시보드
- 한국어 중심 타이포그래피
- 조용하지만 선명한 CTA
- 시험과 학습에 어울리는 절제된 색상
- 사용자가 다음 행동을 바로 알 수 있는 정보 배치

### 4.2 타이포그래피

- 기본 한글 폰트는 Pretendard를 사용합니다.
- 제목은 굵고 명확하게 쓰되, 한국어 가독성을 위해 줄 높이를 너무 낮추지 않습니다.
- 본문은 긴 문장을 읽기 쉽도록 넉넉한 줄 간격을 유지합니다.
- 한국어 문장은 단어 중간에서 끊기지 않게 처리합니다.
- 화면 문구는 번역투가 아니라 자연스러운 한국어 존댓말로 작성합니다.

### 4.3 색상

- 기본 배경은 흰색, 아주 옅은 회색, 깊은 차콜 계열을 화면 목적에 따라 사용합니다.
- 주요 강조색은 한 화면에서 하나만 사용합니다.
- AI 서비스처럼 보이게 하려고 보라색과 파란색 그라데이션에 의존하지 않습니다.
- 학습 진행, 정답, 경고, 오류는 의미가 분명한 색상 체계를 사용합니다.
- 어두운 화면을 쓰는 경우에도 시험 풀이와 긴 글 입력 화면은 읽기 편해야 합니다.

권장 의미 색상:

- 학습 진행: emerald 계열
- 주의 및 마감: amber 계열
- 오류 및 미완료: rose 계열
- 기본 텍스트: zinc 또는 neutral 계열

### 4.4 레이아웃

- 대시보드는 사용자의 현재 상태와 다음 행동이 첫 화면에서 보이도록 구성합니다.
- 동일한 3열 카드 반복만 사용하지 않습니다. 중요한 카드와 보조 카드를 크기와 위치로 구분합니다.
- 쓰기, 문제 풀이, 모의고사는 집중을 방해하지 않도록 작업 영역을 넓게 둡니다.
- AI 튜터는 보조 도구이므로 화면을 가리지 않는 패널이나 플로팅 버튼으로 제공합니다.
- 모바일에서는 모든 비대칭 레이아웃을 한 열로 안전하게 접습니다.

### 4.5 카드와 버튼

- 카드 안에 또 다른 카드를 중첩하지 않습니다.
- 반복 목록의 항목, 피드백 요약, 문제 유형 선택처럼 명확히 묶어야 하는 정보에만 카드를 사용합니다.
- 주요 CTA 버튼은 최소 48px 높이를 보장합니다.
- 버튼에는 hover, active, focus 상태를 모두 제공합니다.
- 중요한 버튼은 아이콘과 텍스트를 함께 쓰되, 아이콘만 있는 버튼은 Tooltip으로 의미를 알려줍니다.

### 4.6 모션

- 모션은 사용자의 현재 위치와 다음 행동을 이해시키는 데만 사용합니다.
- 버튼, 카드 hover, 패널 열림, 섹션 진입에 부드러운 움직임을 적용합니다.
- 시험 타이머, OMR, 답안 입력 화면에는 산만한 장식 모션을 쓰지 않습니다.
- 애니메이션은 `transform`과 `opacity` 중심으로 구현합니다.

## 5. 화면별 구현 기준

### 5.1 공통 레이아웃

사용자에게 보이는 변화:

- 왼쪽 사이드바로 주요 메뉴에 빠르게 접근합니다.
- 사용자 이름, 플랜, 언어 선택, AI 튜터를 항상 찾기 쉬운 위치에 둡니다.

기술 기준:

- `AppShell` 컴포넌트로 사이드바, 상단 영역, 본문, AI 튜터 버튼을 관리합니다.
- 모바일에서는 사이드바를 `Sheet`로 전환합니다.
- 현재 경로와 메뉴 활성 상태는 라우터 정보를 기준으로 표시합니다.
- 언어 선택은 `useUserStore`에 저장합니다.

### 5.2 홈 대시보드

사용자에게 보이는 변화:

- 목표 급수까지 남은 거리, 오늘 해야 할 학습, 약점 영역, 이어하기가 한눈에 보입니다.

기술 기준:

- 학습 요약은 `LearningOverview` 컴포넌트로 구성합니다.
- 읽기, 듣기, 쓰기, 모의고사 진입 카드는 각각 명확한 CTA를 가집니다.
- Supanova의 비대칭 Bento 구성을 적용하되, 모바일에서는 한 열 카드 목록으로 바꿉니다.
- 데이터가 없을 때도 빈 화면 대신 시작 행동을 보여줍니다.

### 5.3 AI 맞춤 문제 생성

사용자에게 보이는 변화:

- 영역, TOPIK 단계, 목표 급수, 문제 유형을 선택하고 바로 문제를 생성합니다.

기술 기준:

- 선택값은 `usePracticeStore`에 저장합니다.
- 필수 선택값이 없으면 생성 버튼을 비활성화합니다.
- 문제 유형 설명은 짧은 미리보기 카드로 보여줍니다.
- 생성 중에는 중복 클릭을 막고 진행 상태를 표시합니다.

### 5.4 문제 풀이

사용자에게 보이는 변화:

- 현재 문제 번호, 지문, 보기, 선택 답, 정답 확인, 이전/다음 이동을 명확히 볼 수 있습니다.

기술 기준:

- 지문 영역과 답안 영역을 분리합니다.
- 선택한 답과 정답 상태는 `usePracticeStore`에 저장합니다.
- 북마크와 다시 보기 액션은 아이콘 버튼으로 제공하고 Tooltip을 붙입니다.
- 모바일에서는 지문과 보기 영역이 위아래로 쌓입니다.

### 5.5 쓰기 집중 연습 설정

사용자에게 보이는 변화:

- 51번부터 54번까지 쓰기 유형과 주제를 선택하고 연습을 시작합니다.

기술 기준:

- 유형 선택 전에는 생성 버튼을 비활성화합니다.
- 선택한 유형에 맞는 CTA 문구를 표시합니다.
- 주제 검색과 추천 주제 칩을 제공합니다.
- 설정 값은 `useWritingStore`에 저장합니다.

### 5.6 쓰기 연습

사용자에게 보이는 변화:

- 문제, 작성 영역, 추천 표현, 필수 어휘, 타이머, 제출 버튼을 한 화면에서 사용합니다.

기술 기준:

- 답안 초안은 자동 저장합니다.
- 자동 저장 상태는 화면 상단 또는 입력창 근처에 작게 표시합니다.
- 긴 답안을 입력해도 레이아웃이 흔들리지 않게 작성 영역의 높이를 안정적으로 관리합니다.
- 제출 전 확인 Dialog를 띄웁니다.

### 5.7 쓰기 보관함과 피드백 상세

사용자에게 보이는 변화:

- 완료된 피드백과 작성 중인 답안을 구분해서 볼 수 있습니다.
- 상세 화면에서는 점수, 총평, 문장별 피드백, 다시 쓰기 행동을 확인합니다.

기술 기준:

- 목록 필터와 정렬 상태는 `useFeedbackStore`에 저장합니다.
- 피드백 상세는 `FeedbackScorePanel`, `FeedbackCommentList`, `RewriteActionPanel`로 분리합니다.
- 다시 쓰기 버튼은 기존 답안과 피드백을 `useWritingStore`에 넘겨 연습 화면으로 이동합니다.

### 5.8 내 서재와 단어장

사용자에게 보이는 변화:

- 저장한 문제와 단어를 빠르게 찾고 복습합니다.

기술 기준:

- 검색, 필터, 정렬을 지원합니다.
- 저장된 항목이 없을 때는 문제 풀이 또는 단어 추가로 이어지는 안내를 제공합니다.
- 단어장은 학습 상태, 난이도, 최근 복습일을 표시합니다.

### 5.9 모의고사 결과와 실전 모의고사

사용자에게 보이는 변화:

- 최근 성적, 약점, 전체 응시 기록, 새 모의고사 시작 흐름이 연결됩니다.
- 시험 화면에서는 타이머와 OMR 답안지를 사용할 수 있습니다.

기술 기준:

- 시험 진행 상태는 `useMockExamStore`에 저장합니다.
- 타이머는 화면 이동에도 안정적으로 유지되어야 합니다.
- OMR 패널은 데스크톱에서는 옆 패널, 모바일에서는 하단 Sheet로 제공합니다.
- 시험 종료 전 확인 Dialog를 제공합니다.

### 5.10 게시판과 프로필 설정

사용자에게 보이는 변화:

- 공지, 이벤트, 프로필, 언어, 플랜 정보를 관리합니다.

기술 기준:

- 게시글 목록은 읽기 쉬운 리스트 중심으로 구성합니다.
- 프로필 설정은 저장 전 변경 여부를 감지합니다.
- 언어 변경은 즉시 UI 문구에 반영합니다.

### 5.11 AI 튜터 패널

사용자에게 보이는 변화:

- 어느 화면에서든 AI 튜터를 열고 질문, 문장 교정, 단어 검색, 알림을 확인합니다.

기술 기준:

- `AiTutorPanel`은 전역 오버레이로 구현합니다.
- 패널 상태는 `useAiTutorStore`에 저장합니다.
- 탭은 홈, 채팅, 알림으로 나눕니다.
- 사용자가 현재 보고 있는 화면 정보를 질문 문맥으로 전달할 수 있게 설계합니다.

## 6. 접근성 기준

접근성은 키보드, 스크린 리더, 저시력 사용자도 서비스를 쓸 수 있게 하는 기준입니다.

- 모든 버튼과 입력 요소는 키보드로 접근할 수 있어야 합니다.
- 포커스 상태는 눈에 보여야 합니다.
- 아이콘만 있는 버튼은 보조 설명을 제공합니다.
- 색상만으로 정답, 오류, 경고를 구분하지 않습니다.
- Dialog와 Sheet는 열릴 때 포커스를 내부로 이동시키고 닫힐 때 원래 위치로 되돌립니다.
- 긴 시험 화면에서도 현재 위치와 남은 시간이 명확해야 합니다.

## 7. 데이터와 오류 처리 기준

- 로딩 중에는 빈 화면 대신 Skeleton 또는 진행 상태를 표시합니다.
- 오류가 발생하면 사용자가 다음에 무엇을 해야 하는지 알려줍니다.
- 네트워크 오류와 입력 오류를 구분해 안내합니다.
- 쓰기 답안과 시험 답안은 사용자가 잃어버리지 않도록 저장 전략을 둡니다.
- 삭제, 제출, 시험 종료처럼 되돌리기 어려운 행동은 확인 단계를 둡니다.

## 8. 구현 완료 기준

각 화면은 다음 조건을 만족해야 완료로 봅니다.

- 사용자가 해야 할 다음 행동이 화면에서 명확합니다.
- 모바일과 데스크톱에서 글자와 버튼이 겹치지 않습니다.
- 주요 데이터가 없을 때의 빈 상태가 준비되어 있습니다.
- 로딩, 오류, 성공 상태가 모두 준비되어 있습니다.
- Zustand 스토어 책임이 기능별로 분리되어 있습니다.
- shadcn/ui 기본 컴포넌트를 재사용하되, TALKPIK AI 전용 컴포넌트로 의미를 부여했습니다.
- Tailwind 클래스가 임의로 흩어지지 않고 공통 토큰과 패턴을 따릅니다.
- Supanova 기준에 따라 한국어 문구가 자연스럽고, 레이아웃이 반복적이지 않으며, CTA가 분명합니다.

## 9. 개발자가 먼저 만들 화면 순서

1. 공통 레이아웃과 사이드바
2. 홈 대시보드
3. AI 맞춤 문제 생성
4. 문제 풀이
5. 쓰기 집중 연습 설정
6. 쓰기 연습
7. 쓰기 보관함과 피드백 상세
8. AI 튜터 패널
9. 모의고사 결과와 실전 모의고사
10. 내 서재, 단어장, 게시판, 프로필 설정

이 순서는 사용자의 핵심 학습 루프를 먼저 완성하기 위한 순서입니다. 먼저 대시보드에서 학습을 시작하고, 문제를 풀고, 쓰기 답안을 제출하고, 피드백을 다시 확인할 수 있어야 합니다.
